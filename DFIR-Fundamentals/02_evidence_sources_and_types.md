# Sources & Types of Digital Evidence

Digital evidence does not exist in a vacuum. To effectively hunt for artifacts, an analyst must understand the distinction between physical hardware hosts and the logical data layers residing within them.

---

## 🔌 Physical Sources (The Hardware Layer)

Physical sources are the tangible assets that contain or transport data. These are generally divided into containers, storage devices, and transmission mediums.

### 1. Containers
Devices that house other individual evidence sources.
* **Computer Systems:** Desktops, laptops, and enterprise servers.
* **Mobile Devices:** Smartphones and tablets. *Note: These are treated distinctly in DFIR because their hardware architecture, security controls, and acquisition tools are entirely different from traditional PCs.*

### 2. Storage Devices
* **Internal Storage (HDD/SSD):** Non-volatile media. Data persists even after the device is powered down.
* **Removable Media:** USB thumb drives, external hard drives, and SD cards (also non-volatile).
* **RAM (Random Access Memory):** Highly volatile media. Data is completely lost or heavily degraded when power is removed. This makes live memory forensics a critical priority before pulling a plug.

### 3. Network Transmissions
Data actively moving across a wire or through the air.
* **Wired Networks:** Ethernet packets traveling across local networks or the internet.
* **Wireless Networks:** Wi-Fi, Bluetooth, and cellular traffic.
* > 💡 **Why Network Evidence Matters:** Captured network traffic is incredibly valuable because it can capture command-and-control (C2) communication or data exfiltration details that an attacker might successfully wipe from the local endpoint logs.

---

## 💻 Non-Physical / Logical Sources (The Data Layer)

Logical sources exist as software, structures, and configurations generated by system activity. They are categorized into **producers** (the applications) and **data sources** (the specific items generated).

### 1. Producers (The Software)
Applications and systems that actively generate digital footprints:
* **Operating Systems (Windows, Linux, macOS):** Generate event logs, configuration trees, and user activity records.
* **Web Servers (Apache, Nginx, IIS):** Record incoming HTTP requests, user-agent strings, and session IDs.
* **Web Browsers:** Track user navigation history, cookies, download logs, and cached content.
* **Email Servers:** Maintain comprehensive communication databases, mail routing headers, and attachments.

### 2. Data Sources (The Artifacts)
The actual targets of a forensic examination:
* **Files & Fragments:** Active files, as well as unallocated space containing deleted file fragments that may be partially recovered.
* **File Systems (NTFS, ext4):** Structures that store metadata *about* files (like creation dates, permissions, and directory structures), rather than just the file contents.
* **The Windows Registry:** A massive database storing system configurations, hardware profiles, user preferences, and recent execution history. *(Crucial target for malware persistence investigations).*
* **Memory Structures:** Active network connections, running processes, and unencrypted passwords sitting in live RAM.

---

## 🗺️ Hierarchical Layers: How They Connect

When identifying evidence, think of it as an onion. You have to peel back the physical layers to reach the specific logical logs you need. Any layer in this chain can serve as a pivot point for your investigation:

```text
📦 Physical Container (e.g., Enterprise Server)
  └── 💾 Physical Storage (e.g., 2TB Internal SSD)
        └── 💽 Operating System (e.g., Windows Server 2022)
              └── 📊 Data Sources (e.g., Event Logs, Registry Hives, MFT)
